# README Style Guide (Weeks 1-10 Analysis)

## H2 Headers (Main Sections)
주요 섹션은 번호와 함께 구성되며, `Friendly Terms`, `Common Pitfalls`, `Diagram` 등은 공통적으로 사용됩니다.

### 1. Introduction & Basics
- `## 0. 미리 알면 좋은 용어 (Friendly Terms)`
- `## 1. 핵심 개념` (또는 `왜 Modern C++인가?`, `프로젝트 개요`)

### 2. Core Content
- `## 2. 핵심 개념 요약` (또는 `구현 상세`)
- `## 3. 아키텍처 설계` (프로젝트 주차)

### 3. Pitfalls & Warnings
- `## 2. 자주 하는 실수 (Common Pitfalls)` (또는 3번)
- `> [!WARNING]`, `> [!DANGER]`, `> [!TIP]` 등의 Alert 문법 사용

### 4. Practice & Guide
- `## 3. 실습 가이드` (또는 4번)
- `## Step-by-Step Guide`
- `## Theory Overview`

### 5. Build & Run
- `## 4. 빌드 및 실행` (또는 5번)
- `## 6. 실행 방법 (터미널)`

### 6. Visualization & Misc
- `## Diagram` (Mermaid)
- `## 7. PowerShell 기초` (1주차 전용)

## H3 Headers (Subsections)
세부 내용은 알파벳(A, B, C) 또는 구체적인 방법론으로 나뉩니다.
모든 주차별 상세 개념 목록은 [CONCEPTS_LIST.md](./CONCEPTS_LIST.md)를 참조하세요.

### Concepts (A, B, C Pattern)
- `### A. [Concept Name]`
- `### B. [Concept Name]`
- `### C. [Concept Name]`
