# 20주차: 멀티쓰레드 서버 아키텍처

"I/O랑 로직 처리를 분리하고 싶어요."
네트워크 입출력(I/O)과 게임 로직(Logic)을 분리하는 **Worker Thread 패턴**을 배웁니다.

## 1. 핵심 구조
- **I/O Thread**: `select`나 `IOCP`를 돌면서 패킷을 받아 큐에 넣습니다.
- **Logic Thread**: 큐에서 패킷을 꺼내 처리하고, 결과를 다시 보냅니다.
- 이렇게 하면 I/O가 느려져도 게임 로직은 멈추지 않습니다.

## 2. 실습 가이드
1. **01_worker_pattern.cpp**: I/O 쓰레드와 로직 쓰레드가 큐를 통해 통신하는 구조.

## 3. 빌드 및 실행
```powershell
.\build_cmake.bat
```
